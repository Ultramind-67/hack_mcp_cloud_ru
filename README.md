# Business AI Agent (MCP Architecture)

Проект реализует автономного бизнес-ассистента на базе **Cloud.ru Evolution (GigaChat)**, использующего протокол **MCP (Model Context Protocol)**. 

Агент способен самостоятельно **исследовать интернет**: искать информацию в Google, читать содержимое веб-страниц, фильтровать данные и формировать аналитические отчеты.

## ✨ Ключевые возможности

1.  **Умный поиск (Google API)**: Поддержка пагинации и поиска конкретных фактов.
2.  **Чтение сайтов (Jina Reader)**: Превращение любого веб-сайта в чистый Markdown для анализа цен, услуг и условий.
3.  **ReAct Архитектура**: Агент работает в цикле *Мысль → Действие → Наблюдение*. Он не отвечает сразу, а проводит исследование шаг за шагом.
4.  **Устойчивость**: Специальный парсер обрабатывает ответы GigaChat, игнорируя технический мусор и извлекая JSON-команды даже из "грязного" текста.

## 🏗 Архитектура

Система разделена на два независимых компонента:

1.  **MCP Server ("Руки")**:
    *   Работает на `fastmcp`.
    *   Запускается как веб-сервер (SSE transport).
    *   Содержит инструменты (`tools/`) для взаимодействия с внешним миром.

2.  **AI Agent ("Мозг")**:
    *   Клиент на Python (`AsyncOpenAI` + `mcp` SDK).
    *   Реализует цикл выполнения (Execution Loop).
    *   Управляет контекстом диалога и принимает решения о вызове инструментов.

### Структура проекта

```text
.
├── agent/                 # 🧠 Клиентская часть (LLM)
│   ├── main.py            # Точка входа (ReAct Loop, парсинг)
│   └── llm_client.py      # Настройки подключения к Cloud.ru
├── mcp_server/            # 🛠 Серверная часть (Инструменты)
│   ├── server.py          # Точка входа сервера
│   ├── mcp_instance.py    # Singleton инстанс FastMCP
│   ├── utils.py           # Утилиты (проверка ENV и т.д.)
│   └── tools/             # 📂 ПАПКА С ИНСТРУМЕНТАМИ
│       ├── __init__.py
│       ├── web_search.py  # Поиск в Google
│       └── jina_reader.py # Скрапинг сайтов (Jina AI)
├── .env                   # API ключи и настройки
└── pyproject.toml         # Зависимости
```

## 🚀 Установка и Настройка

### 1. Подготовка окружения
Требуется Python 3.10+ (рекомендуется 3.11/3.12).

```bash
# Создание venv (Windows)
python -m venv .venv
.venv\Scripts\activate

# Создание venv (Mac/Linux)
python3 -m venv .venv
source .venv/bin/activate

# Установка зависимостей
pip install -r req.txt
```

### 2. Настройка ключей (.env)
Создайте файл `.env` в корне проекта. Для работы требуются ключи от трех сервисов:

```ini
# --- 1. LLM (GigaChat / Cloud.ru) ---
# Получить: https://console.cloud.ru/
API_KEY=AQVN...

# --- 2. Google Search (Поиск) ---
# Получить API Key: https://console.cloud.google.com/ (включить Custom Search API)
GOOGLE_API_KEY=AIza...
# Получить CX ID: https://programmablesearchengine.google.com/ (включить "Search the entire web")
GOOGLE_CSE_ID=012345...

# --- 3. Jina Reader (Чтение сайтов) ---
# Получить бесплатно: https://jina.ai/reader
JINA_API_KEY=jina_...

# Настройки сервера
PORT=8000
HOST=0.0.0.0
```

## ▶️ Запуск

Для работы системы нужно **два** терминала. Запускать команды строго из **корня проекта**.

### Терминал 1: Запуск сервера (MCP)
```bash
python -m mcp_server.server
```
*Ожидаемый вывод:* `🚀 MCP Server running on http://0.0.0.0:8000/sse`

### Терминал 2: Запуск агента (Client)
```bash
python -m agent.main
```
*Ожидаемый вывод:* 
```text
✅ Подключено к MCP-серверу!
🛠 Загружено инструментов: 2
🤖 Агент готов!
```

## 🛠 Разработка

### Как добавить новый инструмент?
Не нужно трогать `server.py` или `main.py`. Просто создайте новый файл в папке `mcp_server/tools/`. Сервер подхватит его автоматически при перезапуске.

**Пример (`mcp_server/tools/calculator.py`):**

```python
from mcp_server.mcp_instance import mcp

@mcp.tool(description="Складывает два числа. Используй для расчетов.")
async def add(a: int, b: int) -> int:
    return a + b
```
